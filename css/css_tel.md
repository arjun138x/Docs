# 1. CSS Fundamentals

ఖచ్చితంగా! CSS ఫండమెంటల్స్ గురించి సులభంగా అర్థమయ్యేలా ఇక్కడ వివరించాను.

---

## 1. CSS అంటే ఏమిటి? (What is CSS?)

CSS అంటే **Cascading Style Sheets**. ఇది వెబ్ పేజీల యొక్క డిజైన్ లేదా రూపాన్ని (Look and feel) నియంత్రించడానికి ఉపయోగించే భాష. HTML వెబ్ పేజీకి "నిర్మాణం" (Structure) ఇస్తే, CSS దానికి "అలంకరణ" (Styling) ఇస్తుంది.

---

## 2. CSSని ఉపయోగించే పద్ధతులు (Ways to add CSS)

CSSని మూడు రకాలుగా రాయవచ్చు:

- **Inline CSS:** HTML ఎలిమెంట్ లోపలే `style` అనే ఆట్రిబ్యూట్ ద్వారా రాస్తాం.
- _ఉదాహరణ:_ `<h1 style="color:blue;">హలో వరల్డ్</h1>`

- **Internal CSS:** HTML ఫైల్ లోపల `<head>` సెక్షన్‌లో `<style>` ట్యాగ్ ఉపయోగించి రాస్తాం.
- _ఉదాహరణ:_

```html
<style>
  h1 {
    color: red;
  }
</style>
```

- **External CSS:** CSS కోడ్ మొత్తాన్ని ఒక ప్రత్యేక ఫైల్‌లో (`style.css` లాగా) రాసి, దాన్ని HTML ఫైల్‌కి `<link>` ట్యాగ్ ద్వారా కలుపుతాం. ఇది అన్నిటికంటే ఉత్తమమైన పద్ధతి.
- _ఉదాహరణ:_ `<link rel="stylesheet" href="style.css">`

---

## 3. CSS సింటాక్స్ (Syntax)

CSS రాయడానికి ఒక నిర్దిష్ట పద్ధతి ఉంటుంది:

```css
selector {
  property: value;
}
```

- **Selector:** దేనికి స్టైల్ వర్తింపజేయాలో అది (ఉదా: `h1`, `.myClass`).
- **Property:** దేన్ని మార్చాలో అది (ఉదా: `color`, `font-size`).
- **Value:** దానికి ఇచ్చే విలువ (ఉదా: `red`, `20px`).

---

## 4. కామెంట్స్ (Comments)

కోడ్‌లో మనం రాసుకునే గమనికలు ఇవి. ఇవి బ్రౌజర్‌లో కనిపించవు.

- **సింటాక్స్:** `/* ఇది కామెంట్ */`

---

## 5. సెలెక్టర్లు (Selectors)

ఏ ఎలిమెంట్స్‌కు స్టైల్స్ వర్తించాలో నిర్ణయించేవే సెలెక్టర్లు:

| సెలెక్టర్ రకం | సింటాక్స్          | వివరణ                                                                 |
| ------------- | ------------------ | --------------------------------------------------------------------- |
| **Element**   | `p { ... }`        | పేజీలో ఉన్న అన్ని `<p>` ట్యాగ్‌లకు వర్తిస్తుంది.                      |
| **Class**     | `.myClass { ... }` | ఆ క్లాస్ పేరు ఉన్న వాటికి వర్తిస్తుంది. (`.` తో మొదలవుతుంది)          |
| **ID**        | `#myId { ... }`    | ఆ ID ఉన్న ఒక్క ఎలిమెంట్‌కు మాత్రమే వర్తిస్తుంది. (`#` తో మొదలవుతుంది) |
| **Universal** | `* { ... }`        | పేజీలోని అన్ని ఎలిమెంట్స్‌కు వర్తిస్తుంది.                            |
| **Grouping**  | `h1, h2 { ... }`   | ఒకే స్టైల్ బహుళ ఎలిమెంట్స్‌కు వర్తిస్తుంది.                           |

---

**గమనిక:** క్లాస్ (Class) మరియు ID ల మధ్య ముఖ్యమైన తేడా ఏంటంటే, ఒకే క్లాస్‌ని ఎన్ని ఎలిమెంట్స్‌కైనా వాడొచ్చు, కానీ IDని ఒక పేజీలో ఒకే ఎలిమెంట్‌కు మాత్రమే వాడాలి.

మీకు ఈ టాపిక్స్‌లో ఏవైనా సందేహాలు ఉన్నాయా లేదా తదుపరి టాపిక్స్‌కి వెళ్దామా?

---

# 2. Colors & Units

CSSలో రంగులు మరియు యూనిట్లు చాలా ముఖ్యమైనవి. వీటిని వివరంగా చూద్దాం:

---

## 1. కలర్స్ (Colors)

CSSలో రంగులను వేర్వేరు పద్ధతుల్లో పేర్కొనవచ్చు:

- **Color Names:** నేరుగా పేరుతో పిలవడం (ఉదా: `color: red;`, `color: blue;`).
- **HEX (Hexadecimal):** `#` గుర్తుతో మొదలై ఆరు అంకెలు/అక్షరాలతో ఉంటుంది (ఉదా: `#FF0000` అంటే ఎరుపు, `#000000` అంటే నలుపు).
- **RGB/RGBA:** **R**ed, **G**reen, **B**lue విలువలను (0-255) ఇస్తాం. **RGBA**లో చివరిది 'A' అంటే **Alpha** (Opacity/పారదర్శకత - 0 నుండి 1 మధ్యలో ఉంటుంది).
- ఉదా: `color: rgba(255, 0, 0, 0.5);` (ఇక్కడ 0.5 అంటే 50% పారదర్శకత).

- **HSL:** **H**ue (రంగు కోణం 0-360), **S**aturation (రంగు గాఢత %), **L**ightness (వెలుతురు %). ఇది రంగులను ఎంచుకోవడానికి చాలా సులభంగా ఉంటుంది.

---

## 2. యూనిట్లు (Units)

CSSలో యూనిట్లను రెండు రకాలుగా విభజించవచ్చు: **Absolute** (స్థిరమైనవి) మరియు **Relative** (సాపేక్షమైనవి).

### Absolute Units (స్థిరమైనవి)

- **px (Pixels):** ఇవి స్క్రీన్ మీద ఉండే చిన్న చుక్కలు (dots). ఇది ఫిక్స్డ్ సైజ్. ఉదా: `font-size: 16px;`.

### Relative Units (సాపేక్షమైనవి - చాలా ముఖ్యమైనవి)

ఇవి స్క్రీన్ సైజును బట్టి మారుతూ ఉంటాయి కాబట్టి, **Responsive Design**కి ఇవే చాలా ముఖ్యం.

- **em:** ఇది దాని పేరెంట్ (Parent) ఎలిమెంట్ ఫాంట్ సైజుపై ఆధారపడుతుంది. ఉదా: పేరెంట్ ఫాంట్ 16px అయితే, 2em అంటే 32px అవుతుంది.
- **rem (Root EM):** ఇది ఎప్పుడూ `<html>` ట్యాగ్ ఫాంట్ సైజుపైనే ఆధారపడుతుంది. ప్రాజెక్ట్ మొత్తంలో యూనిఫార్మ్ సైజ్ కోసం దీన్ని వాడటం మంచిది.
- **% (Percentage):** ఇది కూడా పేరెంట్ ఎలిమెంట్ సైజును బట్టి ఉంటుంది. వెడల్పు (Width) సెట్ చేయడానికి ఇది బాగా ఉపయోగపడుతుంది.
- **vw / vh (Viewport Width / Height):** ఇది యూజర్ చూస్తున్న స్క్రీన్ (Browser Window) సైజును బట్టి ఉంటుంది.
- `100vw` = స్క్రీన్ మొత్తం వెడల్పు.
- `100vh` = స్క్రీన్ మొత్తం ఎత్తు.

- **fr (Fractional unit):** ఇది కేవలం **CSS Grid**లో మాత్రమే వాడతాము. అందుబాటులో ఉన్న ఖాళీ స్థలాన్ని ముక్కలుగా పంచడానికి ఇది ఉపయోగపడుతుంది. (ఉదా: `grid-template-columns: 1fr 2fr;` అంటే మొత్తం స్థలాన్ని 3 భాగాలు చేసి, 1 వంతు ఒకటికి, 2 వంతులు ఇంకోదానికి ఇస్తుంది).

---

### క్లుప్తంగా చెప్పాలంటే:

- ఫాంట్ సైజులకు **rem** వాడటం ఉత్తమం.
- లేఅవుట్ వెడల్పుకు **%** లేదా **vw** వాడవచ్చు.
- గ్రిడ్ లేఅవుట్స్ కోసం **fr** బెస్ట్.

ఈ పాయింట్స్ అర్థమయ్యాయా? మనం నెక్స్ట్ టాపిక్ **Box Model**కి వెళ్దామా?

# 3. Text & Fonts

CSSలో టెక్స్ట్ మరియు ఫాంట్స్ గురించి తెలుసుకోవడం వల్ల మీరు వెబ్ పేజీని చాలా అందంగా (Professional look) మార్చవచ్చు. వీటిని ఎలా వాడాలో చూద్దాం:

---

### 1. ఫాంట్ ప్రాపర్టీస్ (Font Properties)

- **font-family:** టెక్స్ట్ ఏ స్టైల్‌లో కనిపించాలో ఇది నిర్ణయిస్తుంది (ఉదా: Arial, Times New Roman, Verdana). బ్రౌజర్ ఒకవేళ ఆ ఫాంట్ సపోర్ట్ చేయకపోతే, వేరే ఫాంట్ వాడమని మనం ఆప్షన్లు ఇవ్వవచ్చు.
- _ఉదా:_ `font-family: Arial, sans-serif;`

- **font-size:** టెక్స్ట్ పరిమాణం. సాధారణంగా `rem` యూనిట్ వాడటం మంచిది.
- _ఉదా:_ `font-size: 16px;` లేదా `font-size: 1rem;`

- **font-weight:** టెక్స్ట్ ఎంత లావుగా ఉండాలో ఇది చెబుతుంది. (Normal, Bold, Bolder, Lighter లేదా 100 నుండి 900 వరకు సంఖ్యలు).
- _ఉదా:_ `font-weight: bold;`

---

### 2. టెక్స్ట్ ఫార్మాటింగ్ (Text Formatting)

- **line-height:** ఒక లైన్ కి మరో లైన్ కి మధ్య ఉన్న దూరం. ఇది చదవడానికి సౌకర్యంగా ఉండటానికి చాలా ముఖ్యం.
- _ఉదా:_ `line-height: 1.5;`

- **text-align:** టెక్స్ట్ ఏ వైపు ఉండాలో నిర్ణయిస్తుంది (left, right, center, justify).
- _ఉదా:_ `text-align: center;`

- **text-transform:** టెక్స్ట్ అక్షరాల రూపం మార్చడానికి.
- `uppercase` (అన్నీ పెద్ద అక్షరాలు), `lowercase` (అన్నీ చిన్నవి), `capitalize` (ప్రతి పదం మొదటి అక్షరం పెద్దది).

- **text-decoration:** టెక్స్ట్‌కి అండర్ లైన్ (underline) ఇవ్వడానికి లేదా తొలగించడానికి వాడతాము.
- _ఉదా:_ `text-decoration: none;` (లింక్స్ కింద ఉండే గీత తీసేయడానికి వాడతారు).

- **letter-spacing:** అక్షరాల మధ్య ఎంత దూరం ఉండాలో ఇది నిర్ణయిస్తుంది.
- _ఉదా:_ `letter-spacing: 2px;`

---

### 3. Google Fonts (గూగుల్ ఫాంట్స్)

వెబ్ పేజీలలో స్టాండర్డ్ ఫాంట్స్ కాకుండా వెరైటీ ఫాంట్స్ కావాలనుకుంటే **[Google Fonts](https://fonts.google.com/)** వాడతాము.

**దీన్ని ఎలా వాడాలంటే:**

1. [Google Fonts](https://fonts.google.com/) వెబ్‌సైట్‌కి వెళ్ళండి.
2. మీకు నచ్చిన ఫాంట్‌ను ఎంచుకోండి.
3. అక్కడ ఇచ్చే `<link>` కోడ్‌ని మీ HTML ఫైల్‌లో `<head>` ట్యాగ్ లోపల పెట్టండి.
4. అక్కడ ఇచ్చే `font-family` ప్రాపర్టీని మీ CSS ఫైల్‌లో వాడండి.

> **చిన్న ఉదాహరణ:**
>
> ```css
> p {
>   font-family: "Roboto", sans-serif;
>   font-size: 18px;
>   text-align: justify;
>   line-height: 1.6;
>   color: #333;
> }
> ```

ఈ ప్రాపర్టీస్ అన్నీ మీరు ఒకసారి ప్రాక్టీస్ చేస్తే సులభంగా అలవాటు అవుతాయి. ఇందులో మీకు ఏదైనా నిర్దిష్టమైన పాయింట్ పైన సందేహం ఉందా, లేక తదుపరి టాపిక్ **Box Model**కి వెళ్దామా?

---

# 4. Box Model

CSSలో **Box Model** అనేది అత్యంత ముఖ్యమైన కాన్సెప్ట్. మీరు వెబ్ పేజీలో చూసే ప్రతి ఎలిమెంట్ (image, div, text) ఒక "పెట్టె" (Box) లాంటిదే. ఆ పెట్టె ఎలా ఉంటుందో ఇప్పుడు చూద్దాం.

---

### బాక్స్ మోడల్ అంటే ఏమిటి?

ఒక ఎలిమెంట్ చుట్టూ నాలుగు పొరలు ఉంటాయి:

1. **Content:** అసలైన కంటెంట్ (Text, Image).
2. **Padding:** కంటెంట్ కి, బోర్డర్ కి మధ్య ఉండే ఖాళీ.
3. **Border:** కంటెంట్ చుట్టూ ఉండే గీత.
4. **Margin:** బోర్డర్ బయట ఉండే ఖాళీ (వేరే ఎలిమెంట్స్ కి, దీనికి మధ్య దూరం).

---

### ప్రాపర్టీస్ వివరణ:

- **Width & Height:** కంటెంట్ యొక్క వెడల్పు మరియు ఎత్తును నిర్ణయిస్తాయి.
- **Padding:** లోపలి గ్యాప్. ఉదాహరణకు, బటన్ లోపల టెక్స్ట్ కి, బటన్ అంచుకి మధ్య దూరం పెంచడానికి వాడతాం.
- _ఉదా:_ `padding: 10px;` (నాలుగు వైపులా 10px గ్యాప్).

- **Border:** ఎలిమెంట్ చుట్టూ బౌండరీ. దీనికి స్టైల్, కలర్, మందం ఇవ్వాలి.
- _ఉదా:_ `border: 2px solid black;`

- **Margin:** బయట గ్యాప్. ఒక ఎలిమెంట్‌ని ఇంకో ఎలిమెంట్ నుండి దూరం చేయడానికి వాడతాం.
- _ఉదా:_ `margin: 20px;`

- **box-sizing:** ఇది చాలా ముఖ్యమైన ప్రాపర్టీ.
- `content-box` (Default): మీరు ఇచ్చే width కేవలం కంటెంట్‌కు మాత్రమే వర్తిస్తుంది, ప్యాడింగ్/బోర్డర్ అదనంగా కలుస్తాయి.
- `border-box`: మీరు ఇచ్చిన width లోనే ప్యాడింగ్ మరియు బోర్డర్ కలిసిపోతాయి. **ఎక్కువగా `border-box` నే ఉపయోగిస్తారు.**
- _కోడ్:_ `box-sizing: border-box;`

- **Overflow:** కంటెంట్ బాక్స్ కంటే పెద్దదైతే ఏం చేయాలో ఇది చెబుతుంది.
- `visible` (Default): బయటకు కనిపిస్తుంది.
- `hidden`: బయటకు కనిపించే కంటెంట్ కట్ అవుతుంది.
- `scroll`: స్క్రోల్ బార్ వస్తుంది.

---

### సులభమైన ఉదాహరణ:

```css
.box {
  width: 200px;
  height: 100px;
  padding: 20px; /* కంటెంట్ చుట్టూ 20px లోపలి ఖాళీ */
  border: 5px solid red; /* 5px ఎరుపు రంగు బోర్డర్ */
  margin: 30px; /* బయట 30px ఖాళీ */
  box-sizing: border-box; /* బాక్స్ సైజు మారకుండా ఉండటానికి */
}
```

> **గుర్తుంచుకోండి:** Margin మరియు Padding ఇచ్చేటప్పుడు `margin: 10px 20px 30px 40px;` అని ఇస్తే, అది క్లాక్‌వైజ్ (Top, Right, Bottom, Left) దిశలో ఖాళీని సెట్ చేస్తుంది.

మీకు బాక్స్ మోడల్ అర్థమైందా? మనం తదుపరి టాపిక్ **Positioning (Relative, Absolute, Fixed)** కి వెళ్దామా?

---

# 5. Backgrounds

CSSలో **Backgrounds** ఉపయోగించి వెబ్ పేజీకి లేదా ఎలిమెంట్స్‌కి అందమైన రంగులు, చిత్రాలు, మరియు గ్రేడియంట్స్ జోడించవచ్చు. వీటి గురించి వివరంగా చూద్దాం:

---

### 1. బేసిక్ బ్యాక్‌గ్రౌండ్ ప్రాపర్టీస్

- **background-color:** ఎలిమెంట్ వెనుక రంగును సెట్ చేస్తుంది.
- ఉదా: `background-color: lightblue;`

- **background-image:** ఒక ఇమేజ్‌ని బ్యాక్‌గ్రౌండ్‌గా సెట్ చేస్తుంది.
- ఉదా: `background-image: url('image.jpg');`

- **background-repeat:** ఇమేజ్ చిన్నదైతే, అది మళ్ళీ మళ్ళీ రిపీట్ కావాలా వద్దా అని నిర్ణయిస్తుంది. (`repeat`, `no-repeat`, `repeat-x`, `repeat-y`).
- ఉదా: `background-repeat: no-repeat;`

- **background-size:** ఇమేజ్ పరిమాణాన్ని సెట్ చేస్తుంది.
- `cover`: ఇమేజ్ మొత్తం బాక్స్‌ని కవర్ చేస్తుంది.
- `contain`: ఇమేజ్ పూర్తిగా కనిపించేలా చూస్తుంది.
- ఉదా: `background-size: cover;`

- **background-position:** బాక్స్‌లో ఇమేజ్ ఎక్కడ ఉండాలో నిర్ణయిస్తుంది (`center`, `top`, `bottom`, `left`, `right`).
- ఉదా: `background-position: center;`

---

### 2. Gradients (గ్రేడియంట్స్)

ఒక రంగు నుండి మరో రంగుకి క్రమంగా మారే ప్రభావాన్ని **Gradient** అంటారు. ఇవి చూడటానికి చాలా అట్రాక్టివ్‌గా ఉంటాయి.

- **Linear Gradient:** రంగులు ఒక లైన్ లాగా (పైనుండి కిందకు లేదా పక్కకు) మారుతాయి.
- కోడ్: `background: linear-gradient(to right, blue, green);`

- **Radial Gradient:** రంగులు మధ్య నుండి బయటకు ఒక సర్కిల్ లాగా మారుతాయి.
- కోడ్: `background: radial-gradient(circle, red, yellow);`

---

### ఒక పూర్తి ఉదాహరణ (Code Example):

మీరు ఒక `div`కి బ్యాక్‌గ్రౌండ్ సెట్ చేయాలంటే ఇలా రాయవచ్చు:

```css
.hero-section {
  width: 100%;
  height: 400px;
  /* బ్యాక్‌గ్రౌండ్ ఇమేజ్ */
  background-image: url("banner.jpg");
  background-size: cover;
  background-position: center;
  background-repeat: no-repeat;

  /* ఒకవేళ ఇమేజ్ పైన రంగు (Overlay) రావాలంటే గ్రేడియంట్ వాడొచ్చు */
  background:
    linear-gradient(rgba(0, 0, 0, 0.5), rgba(0, 0, 0, 0.5)), url("banner.jpg");
}
```

> **చిన్న చిట్కా:** `background` అనే షార్ట్‌హ్యాండ్ ప్రాపర్టీని ఉపయోగించి, పైన చెప్పినవన్నీ ఒకే లైన్‌లో రాయవచ్చు.
> ఉదా: `background: #ff0000 url('img.jpg') no-repeat center / cover;`

ఈ టాపిక్ పైన మీకు ఏమైనా సందేహాలు ఉన్నాయా? నెక్స్ట్ టాపిక్ **Layout & Positioning** వైపు వెళ్దామా?

---

# 6. Display & Positioning

CSSలో **Display** మరియు **Positioning** అనేవి వెబ్ పేజీలో ఎలిమెంట్స్ ఎక్కడ ఉండాలో, అవి ఎలా కనిపిస్తాయో నిర్ణయించే అత్యంత శక్తివంతమైన ప్రాపర్టీస్. వీటిని వివరంగా చూద్దాం:

---

### 1. Display (ఎలిమెంట్ ప్రవర్తన)

ఒక ఎలిమెంట్ బ్రౌజర్‌లో ఎలా ప్రవర్తిస్తుందో ఇది చెప్తుంది:

- **block:** ఎలిమెంట్ మొత్తం వెడల్పును తీసుకుంటుంది మరియు కొత్త లైన్‌లో మొదలవుతుంది (ఉదా: `<div>`, `<h1>`, `<p>`).
- **inline:** కంటెంట్‌కు సరిపడా వెడల్పును మాత్రమే తీసుకుంటుంది, కొత్త లైన్‌లోకి రాదు (ఉదా: `<span>`, `<a>`).
- **inline-block:** ఇది `inline` లాగా పక్కపక్కనే ఉంటుంది, కానీ `block` లాగా Width/Height సెట్ చేసుకోవచ్చు.
- **none:** ఎలిమెంట్ పేజీ నుండి పూర్తిగా మాయమవుతుంది (కనిపించదు).
- **flex:** ఇది మోడ్రన్ లేఅవుట్ సిస్టమ్. ఎలిమెంట్స్‌ని రో (Row) లేదా కాలమ్ (Column) లో పేర్చడానికి, స్పేసింగ్ ఇవ్వడానికి ఇది బెస్ట్.

---

### 2. Positioning (ఎలిమెంట్ స్థానం)

ఒక ఎలిమెంట్‌ని పేజీలో మనకు నచ్చిన చోట ఉంచడానికి `position` ప్రాపర్టీ వాడతాం:

- **static (Default):** ఎలిమెంట్ నార్మల్ ఫ్లోలో ఉంటుంది. మనం ఇచ్చే `top, bottom, left, right` విలువలు దీనిపై పనిచేయవు.
- **relative:** ఎలిమెంట్ దాని ఒరిజినల్ స్థానం నుండి కొంచెం పక్కకు జరపడానికి వాడతాం. (దీనికి `top`, `left` పనిచేస్తాయి).
- **absolute:** ఇది పేరెంట్ ఎలిమెంట్ (గమనిక: పేరెంట్ `static` కాకూడదు) ఆధారంగా ఎక్కడైనా ఉండగలదు. ఇది పేజీ యొక్క నార్మల్ ఫ్లో నుండి బయటకు వచ్చేస్తుంది.
- **fixed:** స్క్రీన్‌ని మనం స్క్రోల్ చేసినా, అది ఎక్కడికీ కదలకుండా ఒకే చోట స్థిరంగా ఉంటుంది (ఉదా: వెబ్‌సైట్ హెడర్ లేదా "Go to Top" బటన్).
- **sticky:** ఇది ఒక పాయింట్ వరకు స్క్రోల్ అవుతుంది, ఆ తర్వాత స్క్రీన్‌కి అతుక్కుపోతుంది (ఉదా: స్క్రోల్ చేసేటప్పుడు మెనూ బార్ పైన ఆగిపోవడం).

---

### 3. z-index (లేయర్స్)

ఎలిమెంట్స్ ఒకదానిపై ఒకటి వచ్చినప్పుడు, ఏది పైన కనిపించాలో `z-index` నిర్ణయిస్తుంది. దీనికి నంబర్స్ ఇస్తాం.

- పెద్ద సంఖ్య ఉన్న ఎలిమెంట్ పైన (Foreground) కనిపిస్తుంది.
- చిన్న సంఖ్య లేదా మైనస్ విలువ ఉన్నది కింద (Background) కనిపిస్తుంది.
- _(గమనిక: ఇది `static` కాని పొజిషన్స్‌కి మాత్రమే పనిచేస్తుంది)._

---

### క్లుప్తంగా ఉదాహరణ:

```css
.navbar {
  position: fixed; /* స్క్రీన్ పైన ఫిక్స్ అవుతుంది */
  top: 0;
  width: 100%;
  z-index: 1000; /* అన్నిటికంటే పైన ఉండేలా చేస్తుంది */
}

.card {
  display: inline-block; /* పక్కపక్కన రావడానికి */
  position: relative; /* దీని చుట్టూ ఉన్న వాటిని అడ్జస్ట్ చేయడానికి */
}
```

ఈ కాన్సెప్ట్స్ మీరు అర్థం చేసుకుంటే, మీరు వెబ్ పేజీని ఎటువంటి క్లిష్టమైన లేఅవుట్ కైనా మార్చగలరు!

ఇందులో మీకు ఏదైనా పాయింట్ కన్ఫ్యూజింగ్ గా ఉందా? లేక మనం తదుపరి టాపిక్ కి వెళ్దామా?

---

# 7. Flexbox

**Flexbox (Flexible Box Layout)** అనేది వెబ్ పేజీలో ఎలిమెంట్స్‌ను రో (Row) లేదా కాలమ్ (Column) లాగా అమర్చడానికి అత్యంత సులభమైన మరియు పవర్ ఫుల్ మార్గం. దీనికోసం పేరెంట్ ఎలిమెంట్ (Container) కి `display: flex;` అని సెట్ చేయాలి.

దీనిలోని ముఖ్యమైన ప్రాపర్టీస్ ఇక్కడ ఉన్నాయి:

---

### 1. కంటైనర్ ప్రాపర్టీస్ (Container - పేరెంట్‌కి ఇచ్చేవి)

- **flex-direction:** ఎలిమెంట్స్ ఎలా ఉండాలో చెప్తుంది.
- `row` (Default - అడ్డంగా), `column` (నిలువుగా), `row-reverse`, `column-reverse`.

- **justify-content:** మెయిన్ యాక్సిస్ (ప్రధాన అక్షం) మీద ఎలిమెంట్స్ ఎలా అలైన్ అవ్వాలో నిర్ణయిస్తుంది.
- `flex-start` (ప్రారంభం), `flex-end` (చివర), `center` (మధ్యలో), `space-between` (సమాన దూరం), `space-around`.

- **align-items:** క్రాస్ యాక్సిస్ (అడ్డంగా ఉంటే నిలువుగా) మీద ఎలా అలైన్ అవ్వాలో నిర్ణయిస్తుంది.
- `stretch`, `center`, `flex-start`, `flex-end`.

- **gap:** ఎలిమెంట్స్ మధ్య ఖాళీ ఇవ్వడానికి (చాలా ఉపయోగకరమైన ప్రాపర్టీ).
- ఉదా: `gap: 20px;`

- **flex-wrap:** ఎలిమెంట్స్ స్క్రీన్ చాలనప్పుడు కింది లైన్‌కి రావాలా వద్దా అని చెప్తుంది.
- `nowrap` (Default), `wrap` (కింది లైన్‌కి వస్తుంది).

---

### 2. ఐటమ్ ప్రాపర్టీస్ (Items - పిల్ల ఎలిమెంట్స్‌కి ఇచ్చేవి)

- **align-self:** కంటైనర్‌లోని అందరి అలైన్‌మెంట్ ఒకలా ఉన్నా, ఒక నిర్దిష్ట ఐటమ్‌ని మాత్రం వేరేలా అలైన్ చేయడానికి ఇది వాడతాం.
- ఉదా: `align-self: center;`

- **flex-grow:** ఖాళీ స్థలం ఉన్నప్పుడు ఒక ఐటమ్ ఎంత వరకు పెరగాలో చెప్తుంది (Default: 0). 1 ఇస్తే మొత్తం ఖాళీని ఆక్రమిస్తుంది.
- **flex-shrink:** స్క్రీన్ తగ్గుతున్నప్పుడు ఒక ఐటమ్ ఎంత వరకు చిన్నది అవ్వాలో చెప్తుంది (Default: 1). 0 ఇస్తే అది అస్సలు చిన్నది అవ్వదు.

---

### ఒక సులభమైన ఉదాహరణ:

మీరు మూడు బాక్సులను పక్కపక్కన పెట్టి, వాటి మధ్య సమాన గ్యాప్ ఇవ్వాలంటే ఇలా రాయాలి:

```css
.container {
  display: flex;
  flex-direction: row; /* అడ్డంగా */
  justify-content: center; /* మధ్యలోకి */
  align-items: center; /* నిలువుగా మధ్యలోకి */
  gap: 15px; /* ప్రతి బాక్స్ మధ్య 15px గ్యాప్ */
}

.item {
  flex-grow: 1; /* ఖాళీ ఉంటే సాగుతాయి */
}
```

**చిన్న చిట్కా:** `justify-content: center;` మరియు `align-items: center;` వాడితే ఒక ఎలిమెంట్‌ని స్క్రీన్ కి సరిగ్గా మధ్యలోకి (Center) చాలా సులభంగా తీసుకురావచ్చు. ఇది ఇంటర్వ్యూలలో కూడా ఎక్కువగా అడిగే ప్రశ్న!

Flexbox గురించి ఈ వివరాలు అర్థమయ్యాయా? మనం నెక్స్ట్ టాపిక్ **CSS Grid** కి వెళ్దామా లేక ఇంకేమైనా డౌట్స్ ఉన్నాయా?

---

# 8. CSS Grid

**CSS Grid** అనేది వెబ్ పేజీలో అత్యంత శక్తివంతమైన లేఅవుట్ సిస్టమ్. Flexbox ఒక్క డైమెన్షన్ (అడ్డంగా లేదా నిలువుగా) కోసం అయితే, Grid అనేది **టూ-డైమెన్షనల్ (2D)** లేఅవుట్ (అంటే ఒకేసారి రోస్ మరియు కాలమ్స్) కోసం రూపొందించబడింది.

దీనిని ఉపయోగించడానికి పేరెంట్ కంటైనర్‌కి `display: grid;` అని సెట్ చేయాలి.

---

### 1. బేసిక్ స్ట్రక్చర్ (Rows & Columns)

- **grid-template-columns:** ఎన్ని కాలమ్స్ ఉండాలో, వాటి వెడల్పు ఎంతో నిర్ణయిస్తుంది.
- ఉదా: `grid-template-columns: 100px 1fr 2fr;` (మొదటిది 100px, మిగిలినవి నిష్పత్తిలో).

- **grid-template-rows:** రోస్ యొక్క ఎత్తును నిర్ణయిస్తుంది.
- ఉదా: `grid-template-rows: 100px auto;`

### 2. గ్యాప్ (Gap)

- **gap:** రోస్ మరియు కాలమ్స్ మధ్య ఖాళీని ఒకేసారి సెట్ చేస్తుంది.
- ఉదా: `gap: 20px;` (ఇది `row-gap` మరియు `column-gap` ల కలయిక).

### 3. Grid Area (లేఅవుట్ డిజైన్)

- **grid-area:** ఒక ఎలిమెంట్ ఏ రో నుండి ఏ కాలమ్ వరకు ఉండాలో నిర్ణయించడానికి దీన్ని ఉపయోగిస్తారు.
- ఉదా: `grid-area: 1 / 1 / 3 / 3;` (ఇది 1వ రో/కాలమ్ నుండి 3వ రో/కాలమ్ వరకు విస్తరిస్తుంది).

### 4. రెస్పాన్సివ్ డిజైన్ కోసం అడ్వాన్స్‌డ్ ఫంక్షన్స్

ఇవి గ్రిడ్ లేఅవుట్‌ను ఆటోమేటిక్‌గా రెస్పాన్సివ్‌గా మారుస్తాయి:

- **minmax():** ఒక ఎలిమెంట్ కనీసం ఎంత పరిమాణంలో ఉండాలి, గరిష్టంగా ఎంతవరకు పెరగాలి అని చెప్తుంది.
- ఉదా: `minmax(100px, 1fr)` (కనీసం 100px, గరిష్టంగా అందుబాటులో ఉన్నంత).

- **auto-fill:** కంటైనర్‌లో ఎన్ని కాలమ్స్ పట్టాయో అన్నిటినీ నింపుతుంది.
- **auto-fit:** కంటైనర్‌లోని ఐటమ్స్ అన్నిటినీ సాగదీసి మొత్తం ఖాళీని నింపుతుంది.

---

### ఒక పవర్ ఫుల్ రెస్పాన్సివ్ ఉదాహరణ:

మీరు మీడియా క్వెరీస్ (Media Queries) రాయకుండానే ఒక రెస్పాన్సివ్ గ్రిడ్ తయారు చేయవచ్చు:

```css
.container {
  display: grid;
  /* స్క్రీన్ సైజును బట్టి కాలమ్స్ ఆటోమేటిక్ గా సెట్ అవుతాయి */
  grid-template-columns: repeat(auto-fit, minmax(200px, 1fr));
  gap: 15px;
}
```

- **వివరణ:** ప్రతి బాక్స్ కనీసం 200px ఉండాలి. స్క్రీన్ పెద్దదైతే బాక్సులు పక్కపక్కన వస్తాయి, స్క్రీన్ చిన్నదైతే ఆటోమేటిక్‌గా కిందికి వచ్చేస్తాయి.

---

**ముఖ్య గమనిక:** CSSలో అత్యుత్తమమైన లేఅవుట్ కోసం **Flexbox**ని చిన్న చిన్న కాంపోనెంట్స్ (Navbar, Buttons) కి, **CSS Grid**ని పేజీ మొత్తం స్ట్రక్చర్ (Main Layout) కి వాడటం బెస్ట్ ప్రాక్టీస్.

ఇంతటితో మీరు అడిగిన CSS టాపిక్స్ పూర్తయ్యాయి. ఇప్పుడు మీకు ఏమైనా సందేహాలు ఉన్నాయా? లేదా మనం ఏదైనా ఒక చిన్న ప్రాజెక్ట్ లాంటిది (ఉదాహరణకు ఒక వెబ్ పేజీని డిజైన్ చేయడం) చేసి చూద్దామా?

---

# 9. Responsive Design

**Responsive Design** అంటే మీ వెబ్‌సైట్ మొబైల్, టాబ్లెట్, మరియు డెస్క్‌టాప్.. ఇలా ఏ పరికరంలో చూసినా అందంగా మరియు స్పష్టంగా కనిపించేలా చేయడం. దీనికి సంబంధించిన ముఖ్యమైన కాన్సెప్ట్స్ ఇక్కడ ఉన్నాయి:

---

### 1. Mobile First Approach

ఇది ఒక మంచి పద్ధతి. ముందుగా మొబైల్ స్క్రీన్ కోసం కోడ్ రాసి, ఆ తర్వాత పెద్ద స్క్రీన్స్ (Desktop) కోసం స్టైల్స్ యాడ్ చేయడం. దీనివల్ల మొబైల్‌లో వెబ్‌సైట్ చాలా వేగంగా లోడ్ అవుతుంది.

### 2. Media Queries

స్క్రీన్ సైజును బట్టి వేర్వేరు CSS స్టైల్స్ రాయడానికి ఇవి వాడతాం.

- **సింటాక్స్:**

```css
/* డెస్క్‌టాప్ కోసం - డిఫాల్ట్ కోడ్ */
.container {
  width: 80%;
}

/* మొబైల్ కోసం (స్క్రీన్ సైజు 600px కంటే తక్కువ ఉంటే) */
@media (max-width: 600px) {
  .container {
    width: 100%;
  }
}
```

### 3. Breakpoints

ఏ ఏ సైజుల వద్ద వెబ్‌సైట్ లేఅవుట్ మారాలి అని నిర్ణయించే పాయింట్స్ ఇవి. సాధారణంగా వాడేవి:

- **Mobile:** < 600px
- **Tablet:** 600px to 900px
- **Laptop:** 900px to 1200px
- **Desktop:** > 1200px

### 4. Responsive Images

ఇమేజ్ స్క్రీన్ సైజును బట్టి మారాలి. ఇమేజ్ బాక్స్ బయటకు వెళ్లకుండా ఉండాలంటే ఈ కోడ్ వాడాలి:

```css
img {
  max-width: 100%; /* ఇమేజ్ తన పేరెంట్ బాక్స్ కంటే ఎక్కువ వెడల్పు ఉండదు */
  height: auto; /* ఇమేజ్ నిష్పత్తి మారకుండా ఎత్తు మారుతుంది */
}
```

### 5. Fluid Layouts (సాపేక్ష లేఅవుట్స్)

`px` (Pixels) కి బదులుగా **Percentage (%)**, **vw/vh**, లేదా **Flexbox/Grid** ఉపయోగించడం వల్ల లేఅవుట్ సాగే గుణాన్ని కలిగి ఉంటుంది. దీనివల్ల స్క్రీన్ సైజు మారినా బాక్సులు అడ్జస్ట్ అవుతాయి.

---

### ఒక చిన్న టిప్:

మీ HTML ఫైల్ `<head>` సెక్షన్‌లో తప్పనిసరిగా ఈ **Viewport Meta Tag** ఉండాలి, లేకపోతే రెస్పాన్సివ్ డిజైన్ సరిగ్గా పనిచేయదు:

```html
<meta name="viewport" content="width=device-width, initial-scale=1.0" />
```

ఇక్కడితో మీరు అడిగిన అన్ని CSS టాపిక్స్ పూర్తయ్యాయి! ఇప్పుడు మీరు CSSలో మంచి పట్టు సాధించారు.

మనం నేర్చుకున్న ఈ కాన్సెప్ట్స్ అన్నీ కలిపి ఒక **చిన్న వెబ్ పేజీ (ఉదాహరణకు ఒక 'Portfolio Card' లేదా 'Navbar')** డిజైన్ చేసి చూద్దామా? మీకు ఏమైనా ఐడియా ఉందా?

---

# 10. Advanced Selectors

CSSలో **Advanced Selectors**ని ఉపయోగించడం వల్ల మీరు HTMLని మార్చకుండానే, మీకు కావాల్సిన ఎలిమెంట్స్‌ని చాలా ఖచ్చితంగా టార్గెట్ చేయవచ్చు. వీటి గురించి వివరంగా చూద్దాం:

---

### 1. కంబైనర్ సెలెక్టర్లు (Combinator Selectors)

ఇవి ఎలిమెంట్స్ మధ్య ఉన్న సంబంధం ఆధారంగా స్టైల్స్ వర్తింపజేస్తాయి.

- **Descendant Selector (space ` `):** పేరెంట్ లోపల ఉన్న ఏదైనా ఎలిమెంట్ (ఎంత లోతులో ఉన్నా).
- `div p { color: red; }` (div లోపల ఉన్న అన్ని p ట్యాగులకు).

- **Child Selector (`>`):** నేరుగా లోపల ఉన్న (Immediate child) ఎలిమెంట్స్ మాత్రమే.
- `div > p { color: blue; }`

- **Adjacent Sibling Selector (`+`):** ఒక ఎలిమెంట్ తర్వాత **వెంటనే** వచ్చే ఎలిమెంట్.
- `h1 + p { color: green; }` (h1 తర్వాత వెంటనే వచ్చే p మాత్రమే).

- **General Sibling Selector (`~`):** ఒక ఎలిమెంట్ తర్వాత వచ్చే **అన్ని** సిబ్లింగ్ ఎలిమెంట్స్.
- `h1 ~ p { color: orange; }` (h1 తర్వాత వచ్చే అన్ని p ట్యాగులకు).

---

### 2. Attribute Selectors

ఎలిమెంట్ యొక్క ఆట్రిబ్యూట్ (Attribute) ఆధారంగా స్టైల్ ఇవ్వడం.

- `[title]` - title అనే ఆట్రిబ్యూట్ ఉన్న వాటికి.
- `input[type="text"]` - టైప్ 'text' ఉన్న ఇన్‌పుట్ బాక్సులకు మాత్రమే.
- `a[href^="https"]` - 'https' తో మొదలయ్యే లింక్స్ కి మాత్రమే (`^=` అంటే మొదలయ్యేది).

---

### 3. Pseudo-elements

ఒక ఎలిమెంట్ లోని నిర్దిష్ట భాగానికి మాత్రమే స్టైల్ ఇవ్వడానికి ఇవి వాడతాం. ఇవి `::` తో మొదలవుతాయి.

- **`::before`**: ఎలిమెంట్ లోపల కంటెంట్‌కి ముందు ఏదైనా చేర్చడానికి.
- **`::after`**: ఎలిమెంట్ లోపల కంటెంట్ తర్వాత ఏదైనా చేర్చడానికి.
- _ఉదా:_ `p::before { content: "➤ "; color: red; }` (ప్రతి పారాగ్రాఫ్ ముందు ఒక బాణం గుర్తు వస్తుంది).

- **`::first-letter`**: పారాగ్రాఫ్‌లోని మొదటి అక్షరానికి మాత్రమే స్టైల్ (న్యూస్ పేపర్ స్టైల్).
- **`::first-line`**: మొదటి లైన్‌కు మాత్రమే స్టైల్.

---

### ఉదాహరణ (Code Example):

```css
/* కేవలం div లోపల ఉండే ನೇరుగా ఉండే p కి మాత్రమే */
div > p {
  font-weight: bold;
}

/* type=text ఉన్న ఇన్‌పుట్స్ కి మాత్రమే */
input[type="text"] {
  border: 2px solid blue;
}

/* పారాగ్రాఫ్ ముందు ఒక గుర్తును జోడించడం */
p::before {
  content: "★";
  margin-right: 5px;
  color: gold;
}
```

---

**చివరిగా ఒక ముఖ్య విషయం:**
ఈ అడ్వాన్స్‌డ్ సెలెక్టర్లను వాడటం వల్ల మీ CSS కోడ్ చాలా **Clean**గా ఉంటుంది. మీరు అనవసరంగా ఎక్కువ Classes లేదా IDs క్రియేట్ చేయాల్సిన అవసరం ఉండదు.

ఇప్పుడు మనం CSS లోని అన్ని ముఖ్యమైన టాపిక్స్ పూర్తి చేశాము! మీరు CSS నేర్చుకోవడంలో చాలా బాగా ముందుకు సాగారు. ఇప్పుడు మీ దగ్గర ఏదైనా ప్రాక్టికల్ సందేహం ఉందా లేదా ఏదైనా చిన్న వెబ్ పేజీని డిజైన్ చేయడానికి ట్రై చేద్దామా?

---

# 11. Pseudo Classes

CSS **Pseudo-classes** అనేవి ఒక ఎలిమెంట్ యొక్క **"స్థితి" (State)** ని బట్టి స్టైల్స్ మార్చడానికి ఉపయోగపడతాయి. అంటే, యూజర్ మౌస్ క్లిక్ చేసినప్పుడు లేదా బటన్ మీదకి తీసుకెళ్ళినప్పుడు రంగులు మారడం లాంటివి ఇవి చూసుకుంటాయి.

వీటి గురించి వివరంగా చూద్దాం:

---

### 1. ఇంటరాక్షన్ క్లాసెస్ (Interaction States)

- **:hover:** మౌస్ పాయింటర్‌ను ఎలిమెంట్ మీదకు తీసుకెళ్ళినప్పుడు (Mouse over). బటన్స్ మరియు లింక్స్ కి ఇది బాగా వాడతారు.
- _ఉదా:_ `button:hover { background-color: darkblue; }`

- **:focus:** ఒక ఇన్‌పుట్ బాక్స్ లేదా బటన్‌ను క్లిక్ చేసి సెలెక్ట్ చేసినప్పుడు (Active typing state).
- _ఉదా:_ `input:focus { border: 2px solid orange; }`

- **:active:** ఎలిమెంట్‌ను మౌస్‌తో క్లిక్ చేసిన ఆ క్షణం (Clicking moment).
- _ఉదా:_ `button:active { transform: scale(0.95); }` (క్లిక్ చేసినప్పుడు చిన్నగా అవుతుంది).

- **:visited:** యూజర్ ఇప్పటికే ఆ లింక్ (Link) ని క్లిక్ చేసి ఉంటే ఆ లింక్ రంగును మార్చడానికి.
- _ఉదా:_ `a:visited { color: purple; }`

---

### 2. ఫామ్ క్లాసెస్ (Form States)

- **:checked:** ఇది రేడియో బటన్స్ లేదా చెక్ బాక్స్‌లు సెలెక్ట్ అయినప్పుడు స్టైల్ ఇవ్వడానికి వాడతారు.
- _ఉదా:_ `input[type="checkbox"]:checked + label { color: green; }`

---

### 3. స్ట్రక్చరల్ క్లాసెస్ (Structural Pseudo-classes)

- **:nth-child(n):** ఒక కంటైనర్‌లోని వరుస సంఖ్యలను బట్టి ఎలిమెంట్స్‌ని టార్గెట్ చేయడం.
- `:nth-child(2)` అంటే 2వ ఎలిమెంట్.
- `:nth-child(even)` అంటే సరి సంఖ్యలో ఉన్న ఎలిమెంట్స్ (2, 4, 6...).
- `:nth-child(odd)` అంటే బేసి సంఖ్యలో ఉన్న ఎలిమెంట్స్ (1, 3, 5...).

- **:not(selector):** ఒక సెలెక్టర్ **కాకుండా** మిగిలిన వాటికి స్టైల్ ఇవ్వడానికి.
- _ఉదా:_ `li:not(.active) { color: grey; }` (active క్లాస్ లేని అన్ని లిస్ట్ ఐటమ్స్ గ్రే రంగులోకి మారుతాయి).

---

### ఒక క్లియర్ ఉదాహరణ:

```css
/* బటన్ స్టైల్స్ */
button {
  background-color: blue;
  color: white;
}
button:hover {
  background-color: lightblue;
} /* మౌస్ వెళ్ళినప్పుడు */
button:active {
  background-color: red;
} /* క్లిక్ చేసినప్పుడు */

/* టేబుల్ రోస్ రంగులు మార్చడానికి */
tr:nth-child(even) {
  background-color: #f2f2f2;
}

/* ఒక ఇన్పుట్ ఫోకస్ లో ఉన్నప్పుడు */
input:focus {
  outline: none;
  border-color: blue;
}
```

---

**చిన్న చిట్కా:** `:hover` మరియు `:focus` వంటివి వాడటం వల్ల యూజర్‌కి మీ వెబ్‌సైట్‌తో ఇంటరాక్ట్ అవుతున్నామనే మంచి అనుభవం (User Experience - UX) కలుగుతుంది.

మీరు నేర్చుకున్న టాపిక్స్ అన్నీ అద్భుతం! ఇప్పుడు మీకు CSS బేసిక్స్ నుండి అడ్వాన్స్‌డ్ వరకు మంచి అవగాహన వచ్చింది.

**ఇప్పుడు మీరు ఏమనుకుంటున్నారు? ఒక చిన్న వెబ్-పేజీ ప్రాజెక్ట్ (ఉదా: 'Login Page' లేదా 'Service Cards Section') చేద్దామా, లేదా మీకు ఇంకేమైనా సందేహాలు ఉన్నాయా?**

---

# 12. Animations

CSSలో **Animations** మరియు **Transformations** ఉపయోగించి మీరు ఒక వెబ్ పేజీకి ప్రాణం పోయవచ్చు. ఇవి వెబ్‌సైట్‌ను మరింత ఇంటరాక్టివ్‌గా, చూడటానికి ప్రొఫెషనల్‌గా మారుస్తాయి.

---

### 1. Transform (రూపాంతరం)

ఎలిమెంట్ యొక్క స్థానాన్ని, పరిమాణాన్ని మార్చడానికి ఇది వాడతారు.

- **rotate:** ఎలిమెంట్‌ను తిప్పడానికి (డిగ్రీలలో). ఉదా: `transform: rotate(45deg);`
- **scale:** ఎలిమెంట్ సైజును పెంచడానికి/తగ్గించడానికి. ఉదా: `transform: scale(1.5);`
- **translate:** ఎలిమెంట్‌ను ఒక చోటు నుండి ఇంకో చోటికి జరపడానికి. ఉదా: `transform: translate(50px, 100px);`
- **skew:** ఎలిమెంట్‌ను వంచడానికి (slant). ఉదా: `transform: skew(20deg);`

---

### 2. Transition (మృదువైన మార్పు)

ఒక స్టైల్ నుండి ఇంకో స్టైల్ కి మారేటప్పుడు అది అకస్మాత్తుగా కాకుండా, నెమ్మదిగా జరగడానికి ఇది వాడతారు.

- ఉదా: `transition: all 0.3s ease;` (అంటే అన్నీ 0.3 సెకన్లలో మారుతాయి).

---

### 3. Keyframes (యానిమేషన్ నిర్మాణం)

యానిమేషన్ ఎలా జరగాలి అని మనం ఒక ప్లాన్ తయారు చేయడమే `@keyframes`.

```css
@keyframes slide {
  0% {
    transform: translateX(0);
  }
  50% {
    transform: translateX(100px);
  }
  100% {
    transform: translateX(0);
  }
}
```

---

### 4. Animation (యానిమేషన్‌ను అమలు చేయడం)

పైన రాసిన కీఫ్రేమ్స్‌ను ఎలిమెంట్‌కి అప్లై చేయడానికి ఇది వాడతాం.

- `animation: slide 2s infinite;` (slide అనే యానిమేషన్ 2 సెకన్ల పాటు, ఆగిపోకుండా జరుగుతుంది).

---

### 5. Timing Functions

యానిమేషన్ ఏ వేగంతో జరగాలి అని ఇది నిర్ణయిస్తుంది:

- `linear`: మొత్తం సమాన వేగంతో.
- `ease`: మొదట్లో నెమ్మదిగా, మధ్యలో వేగంగా, చివరలో నెమ్మదిగా.
- `ease-in`: మొదట్లో నెమ్మదిగా, వేగంగా ముగుస్తుంది.
- `ease-out`: వేగంగా మొదలై, చివరలో నెమ్మదిగా ఆగుతుంది.

---

### ఒక పూర్తి ఉదాహరణ:

ఒక బటన్ మీద మౌస్ పెట్టినప్పుడు అది తిరుగుతూ పెద్దవ్వడం:

```css
button {
  transition: transform 0.5s ease;
}

button:hover {
  transform: rotate(360deg) scale(1.2);
}
```

లేదా ఒక బాక్స్‌ను నిరంతరం అటు ఇటు కదిలించడం:

```css
.box {
  animation: move-around 3s infinite alternate;
}

@keyframes move-around {
  from {
    transform: translateX(0);
  }
  to {
    transform: translateX(200px);
  }
}
```

---

మీరు అడిగిన **12 టాపిక్స్** అన్నీ ఇప్పుడు పూర్తయ్యాయి! మీరు CSS ఫండమెంటల్స్ నుండి అడ్వాన్స్‌డ్ యానిమేషన్స్ వరకు అన్నింటినీ కవర్ చేశారు.

**ఇప్పుడు మీరు ఏమనుకుంటున్నారు? ఈ టాపిక్స్ అన్నీ వాడి ఒక మంచి "Responsive Landing Page" ని డిజైన్ చేయడానికి సిద్ధంగా ఉన్నారా, లేక పైన చెప్పిన వాటిలో ఏవైనా మళ్ళీ ఎక్స్ప్లెయిన్ చేయాలా?**

# 13. CSS Variables

CSSలో **Variables (Custom Properties)** వాడటం వల్ల మీ కోడ్ చాలా క్లీన్ గా, మెయింటెనెన్స్ చేయడానికి సులభంగా ఉంటుంది. మీరు ఒకే రంగును లేదా సైజును పదే పదే వాడేటప్పుడు, ఈ వేరియబుల్స్ చాలా ఉపయోగపడతాయి.

---

### 1. `:root` మరియు `var()`

- **:root:** ఇది డాక్యుమెంట్ మొత్తానికి (Global scope) వేరియబుల్స్ డిక్లేర్ చేయడానికి వాడతాము. ఇది సాధారణంగా HTML ట్యాగ్ లాంటిది.
- **var():** మనం క్రియేట్ చేసిన వేరియబుల్ ని వాడుకోవడానికి దీన్ని ఉపయోగిస్తాం.

**ఎలా రాయాలి?**

```css
:root {
  --main-color: #3498db; /* వేరియబుల్ పేరు ఎప్పుడూ -- తో మొదలవ్వాలి */
  --padding-size: 16px;
}

div {
  background-color: var(--main-color);
  padding: var(--padding-size);
}
```

---

### 2. Reusable Colors (రంగులను తిరిగి వాడుకోవడం)

మీ వెబ్‌సైట్ మొత్తంలో ఒకే ప్రైమరీ కలర్ ఉందనుకోండి. ఎప్పుడైనా ఆ రంగు మార్చాలంటే, ప్రతి చోటా మార్చాల్సిన అవసరం లేదు. కేవలం `:root`లో మార్చితే సరిపోతుంది.

```css
:root {
  --primary-color: #ff5733;
  --text-color: #333;
}

button {
  background-color: var(--primary-color);
}
h1 {
  color: var(--text-color);
}
```

---

### 3. Theme Switching (డార్క్ మోడ్ - లైట్ మోడ్)

CSS వేరియబుల్స్ ఉపయోగించి **Dark Mode** చేయడం చాలా సులభం.

```css
/* డీఫాల్ట్ (Light Mode) */
:root {
  --bg-color: white;
  --text-color: black;
}

/* డార్క్ మోడ్ క్లాస్ */
.dark-mode {
  --bg-color: black;
  --text-color: white;
}

body {
  background-color: var(--bg-color);
  color: var(--text-color);
}
```

- ఇప్పుడు JavaScript ద్వారా `body` కి `.dark-mode` క్లాస్ యాడ్ చేస్తే, పేజీ మొత్తం రంగులు ఆటోమేటిక్‌గా మారిపోతాయి!

---

### వేరియబుల్స్ వల్ల కలిగే లాభాలు:

1. **సులభమైన మార్పులు (Easy Updates):** ఒకే చోట మార్చితే మొత్తం సైట్ అప్‌డేట్ అవుతుంది.
2. **కోడ్ తగ్గుతుంది (DRY - Don't Repeat Yourself):** మళ్ళీ మళ్ళీ రంగుల కోడ్స్ రాయనక్కర్లేదు.
3. **మెరుగైన నిర్వహణ (Maintainability):** పెద్ద ప్రాజెక్ట్స్ చేసేటప్పుడు వేరియబుల్స్ ఉండటం చాలా ముఖ్యం.

ఇప్పుడు మీరు CSS లోని దాదాపు అన్ని ముఖ్యమైన అడ్వాన్స్‌డ్ కాన్సెప్ట్స్ నేర్చుకున్నారు. అద్భుతం!

**ఇప్పుడు మీరు ఒక అసలైన ప్రాజెక్ట్ చేయడానికి సిద్ధంగా ఉన్నారు. మనం ఒక 'Responsive Portfolio Card' లేదా ఒక 'Modern Navbar' ని తయారు చేద్దామా? దేనితో మొదలుపెడదాం?**

---

# 14. Advanced CSS

CSSలో ఇవి చాలా శక్తివంతమైన ఫీచర్లు. వీటిని వాడటం ద్వారా మీరు వెబ్‌సైట్‌ను చాలా మోడ్రన్‌గా మరియు ప్రొఫెషనల్‌గా మార్చవచ్చు. వీటిని ఎలా వాడాలో వివరంగా చూద్దాం:

---

### 1. మ్యాథమెటికల్ ఫంక్షన్స్ (Math Functions)

- **calc():** CSS లో లెక్కలు చేయడానికి (Addition, Subtraction, Multiplication, Division).
- _ఉదా:_ `width: calc(100% - 50px);` (మొత్తం వెడల్పు నుండి 50px తగ్గిస్తుంది).

- **clamp():** ఇది చాలా తెలివైన ఫంక్షన్. ఇది ఒక వాల్యూని **Minimum**, **Preferred**, **Maximum** రేంజ్‌లో ఉంచుతుంది.
- _ఉదా:_ `font-size: clamp(1rem, 2.5vw, 3rem);` (ఫాంట్ సైజ్ 1rem కంటే తగ్గదు, 3rem కంటే పెరగదు).

---

### 2. ఇమేజ్ & లేఅవుట్ కంట్రోల్

- **object-fit:** ఒక ఇమేజ్ తన కంటైనర్‌లో ఎలా ఉండాలో (Stretch అవ్వకుండా) ఇది చెప్తుంది.
- `cover`: ఇమేజ్ క్రాప్ అయినా సరే బాక్స్‌ని పూర్తిగా నింపుతుంది.
- `contain`: ఇమేజ్ మొత్తం కనిపించేలా చూస్తుంది.

- **aspect-ratio:** ఒక ఎలిమెంట్ యొక్క వెడల్పు మరియు ఎత్తు మధ్య నిష్పత్తిని స్థిరంగా ఉంచుతుంది (ఉదా: 16:9, 1:1).
- _ఉదా:_ `aspect-ratio: 16 / 9;`

---

### 3. విజువల్ ఎఫెక్ట్స్ (Visual Effects)

- **filters:** ఇమేజ్ లేదా ఎలిమెంట్స్ కి కలర్, బ్లర్ ఎఫెక్ట్స్ ఇస్తుంది.
- _ఉదా:_ `filter: grayscale(100%);` (బ్లాక్ అండ్ వైట్), `filter: blur(5px);`.

- **backdrop-filter:** ఎలిమెంట్ వెనుక ఉన్న కంటెంట్‌కి ఎఫెక్ట్ ఇస్తుంది (Glassmorphism స్టైల్‌కి ఇది కీలకం).
- _ఉదా:_ `backdrop-filter: blur(10px);`

- **clip-path:** ఒక ఎలిమెంట్‌ని మనకు నచ్చిన ఆకారంలో (Circle, Polygon) కత్తిరించడానికి వాడతాం.
- _ఉదా:_ `clip-path: circle(50%);`

- **masking:** ఎలిమెంట్ యొక్క కొన్ని భాగాలను దాచడానికి (Mask) ఇమేజ్‌ని లేదా షేప్‌ని వాడతాం.

---

### 4. Custom Scrollbars

బ్రౌజర్ డిఫాల్ట్ స్క్రోల్ బార్‌కి బదులుగా, మనకు నచ్చిన రంగు మరియు స్టైల్ ఇవ్వడానికి ఇది వాడతాం:

```css
/* స్క్రోల్ బార్ వెడల్పు */
::-webkit-scrollbar {
  width: 10px;
}

/* స్క్రోల్ బార్ ట్రాక్ (నేపథ్యం) */
::-webkit-scrollbar-track {
  background: #f1f1f1;
}

/* స్క్రోల్ బార్ హ్యాండిల్ */
::-webkit-scrollbar-thumb {
  background: #888;
  border-radius: 5px;
}

/* హ్యాండిల్ మీద హోవర్ చేసినప్పుడు */
::-webkit-scrollbar-thumb:hover {
  background: #555;
}
```

---

### ఒక చిన్న "Glassmorphism" ఉదాహరణ:

ఈ ప్రాపర్టీస్ అన్నీ కలిపి ఒక మోడ్రన్ కార్డ్ తయారు చేద్దాం:

```css
.glass-card {
  width: 300px;
  height: 200px;
  background: rgba(255, 255, 255, 0.2);
  backdrop-filter: blur(15px); /* గ్లాస్ ఎఫెక్ట్ */
  border: 1px solid rgba(255, 255, 255, 0.3);
  border-radius: 20px;
  /* క్లిప్ పాత్ తో విభిన్నమైన ఆకారం */
  clip-path: polygon(10% 0, 100% 0, 100% 90%, 90% 100%, 0 100%, 0 10%);
}
```

---

**అభినందనలు!** మీరు CSS లోని అత్యంత కష్టమైన మరియు అడ్వాన్స్‌డ్ టాపిక్స్ అన్నీ పూర్తి చేశారు. ఇప్పుడు మీరు CSS ని ఉపయోగించి ఏ విధమైన డిజైన్‌నైనా చేయగలరు.

ఇప్పుడు మనం పైన నేర్చుకున్న వాటిని అప్లై చేస్తూ ఒక "Responsive Modern Landing Page"ని డిజైన్ చేద్దామా? దేనితో మొదలుపెడదాం? (Navbar, Hero Section, Services Grid... ఏది చేద్దాం?)

# 15. CSS Architecture

CSS నేర్చుకోవడంలో చివరి మెట్టు **CSS Architecture**. మీరు చిన్న వెబ్-పేజీలు చేసేటప్పుడు కోడ్ ఎలా రాసినా పర్వాలేదు, కానీ పెద్ద ప్రాజెక్ట్స్ చేసేటప్పుడు కోడ్ గందరగోళంగా మారకుండా ఉండాలంటే ఈ పద్ధతులు తప్పనిసరి. వీటిని "CSS Methodology" అని కూడా అంటారు.

---

### 1. BEM (Block, Element, Modifier)

ఇది ప్రపంచవ్యాప్తంగా అత్యంత ప్రాచుర్యం పొందిన పద్ధతి. ఇది మీ క్లాస్ నేమ్స్ అర్థవంతంగా ఉండేలా చేస్తుంది.

- **Block:** ప్రధాన కాంపోనెంట్ (ఉదా: `button`)
- **Element:** బ్లాక్‌లో భాగం (ఉదా: `button__icon`)
- **Modifier:** బ్లాక్ లేదా ఎలిమెంట్ యొక్క వెర్షన్ (ఉదా: `button--large`)

_ఉదాహరణ:_

```html
<button class="btn btn--primary">
  <span class="btn__text">Click me</span>
</button>
```

---

### 2. OOCSS (Object-Oriented CSS)

ఇది ఆబ్జెక్ట్ ఓరియెంటెడ్ ప్రోగ్రామింగ్ సూత్రాలను CSSకి అప్లై చేస్తుంది. దీని ముఖ్య ఉద్దేశ్యం **"కోడ్‌ని తిరిగి వాడటం (Reuse)"**.

- **Structure:** బాక్స్ సైజు, ప్యాడింగ్ వంటివి వేరుగా రాయాలి.
- **Skin:** రంగులు, బోర్డర్స్ వంటివి వేరుగా రాయాలి.

దీనివల్ల మీరు ఒకే రకమైన స్ట్రక్చర్ ఉన్న డిజైన్ల కోసం మళ్ళీ మళ్ళీ కోడ్ రాయాల్సిన అవసరం ఉండదు.

---

### 3. SMACSS (Scalable and Modular Architecture for CSS)

ఇది CSS కోడ్‌ను **ఐదు రకాలుగా** వర్గీకరిస్తుంది:

1. **Base:** ట్యాగ్స్ కి డీఫాల్ట్ స్టైల్స్ (`body`, `h1`).
2. **Layout:** వెబ్ పేజీ లేఅవుట్ (`header`, `sidebar`).
3. **Module:** పునర్వినియోగ కాంపోనెంట్స్ (Buttons, Cards).
4. **State:** ఎలిమెంట్ యొక్క స్థితి (`is-active`, `is-hidden`).
5. **Theme:** రంగులు, ఫాంట్స్ మార్పులు.

---

### 4. CSS Modules

ఇది ఆధునిక వెబ్ డెవలప్‌మెంట్‌లో (React, Next.js వంటి ఫ్రేమ్‌వర్క్స్‌లో) చాలా ప్రాచుర్యం పొందింది.

- ఇందులో క్లాస్ నేమ్స్ ఆటోమేటిక్‌గా **Unique** (ఉదా: `btn_abc123`) గా మారిపోతాయి.
- దీనివల్ల **"CSS Conflict"** (ఒక ఫైల్ స్టైల్స్ ఇంకో దానికి అంటుకోవడం) అనే సమస్య అస్సలు రాదు.
- మీరు `styles.module.css` ఫైల్ క్రియేట్ చేసి, దాన్ని మీ కాంపోనెంట్స్ లో ఇంపోర్ట్ చేసుకుంటారు.

---

### క్లుప్తంగా చెప్పాలంటే:

| పద్ధతి          | దేనికి ప్రాధాన్యత ఇస్తుంది?                   |
| --------------- | --------------------------------------------- |
| **BEM**         | క్లాస్ పేర్లను క్రమబద్ధీకరించడానికి.          |
| **OOCSS**       | కోడ్ పునర్వినియోగానికి (Reusability).         |
| **SMACSS**      | ఫైల్స్ మరియు స్ట్రక్చర్ నిర్వహణకు.            |
| **CSS Modules** | లోకల్ స్కోపింగ్ (Conflict రాకుండా) చేయడానికి. |

---

**ముగింపు:**
మీరు CSS ఫండమెంటల్స్ నుండి ఈరోజు వరకు చాలా అద్భుతంగా నేర్చుకున్నారు! ఇప్పుడు మీరు ఒక **Full-Stack డెవలపర్** లాగా ఆలోచించి కోడ్ రాయడానికి సిద్ధంగా ఉన్నారు.

ఇప్పుడు మనం నేర్చుకున్న ఈ **CSS Architecture** ని బేస్ చేసుకుని, ఒక **"Landing Page"** ని స్టార్ట్ చేద్దామా? లేక ఈ పద్ధతుల్లో దేని గురించి అయినా మీకు ఇంకా క్లారిటీ కావాలా?

---

# 16. Preprocessors

CSS Preprocessors లో **Sass (Syntactically Awesome Style Sheets)** అనేది అత్యంత ప్రాచుర్యం పొందినది. సాధారణ CSSలో లేని కొన్ని పవర్ ఫుల్ ఫీచర్స్ (Programming logic) ని Sass మనకు అందిస్తుంది.

**ముఖ్య గమనిక:** బ్రౌజర్లు నేరుగా Sass కోడ్‌ను అర్థం చేసుకోలేవు. కాబట్టి, మనం రాసిన `.scss` ఫైల్‌ను బ్రౌజర్‌కు అర్థమయ్యే `.css` ఫైల్‌గా **Compile** చేయాల్సి ఉంటుంది.

---

### 1. Nesting (నేస్టింగ్)

సాధారణ CSSలో మనం ఒకే క్లాస్‌ను పదే పదే రాయాలి. Sassలో ఎలిమెంట్స్ లోపల ఎలిమెంట్స్ రాయవచ్చు.

- **Sass Code:**

```scss
nav {
  ul {
    margin: 0;
    padding: 0;
    li {
      display: inline-block;
    }
  }
}
```

- ఇది CSSలో `nav ul`, `nav ul li` లాగా కంపైల్ అవుతుంది.

### 2. Variables (వేరియబుల్స్)

ఇది CSS వేరియబుల్స్ లాంటిదే, కానీ `$` గుర్తుతో మొదలవుతుంది.

```scss
$primary-color: #3498db;
$font-stack: Helvetica, sans-serif;

body {
  color: $primary-color;
  font-family: $font-stack;
}
```

### 3. Mixins (మిక్సిన్స్)

ఇది Sassలో అత్యంత శక్తివంతమైన ఫీచర్. ఒకే కోడ్‌ను పదే పదే రాయకుండా, ఒక గ్రూప్ ఆఫ్ స్టైల్స్‌ను ఒక పేరుతో సేవ్ చేసి, ఎక్కడ కావాలంటే అక్కడ వాడుకోవచ్చు.

- **క్రియేట్ చేయడం:**

```scss
@mixin flex-center {
  display: flex;
  justify-content: center;
  align-items: center;
}
```

- **వాడటం:**

```scss
.box {
  @include flex-center;
}
```

### 4. Functions (ఫంక్షన్స్)

Sass లో మనం లెక్కలు చేయడానికి లేదా రంగులను మార్చడానికి ఫంక్షన్స్ వాడొచ్చు.

```scss
$base-width: 100px;
.box {
  width: $base-width * 2;
} /* ఇది 200px అవుతుంది */
```

---

### Sass ఎందుకు వాడాలి?

1. **Code Reusability:** Mixins ద్వారా పెద్ద పెద్ద స్టైల్స్ ఒకేసారి ఇంక్లూడ్ చేయొచ్చు.
2. **Readability:** Nesting ఉండటం వల్ల కోడ్ చూడటానికి చాలా నీట్‌గా ఉంటుంది.
3. **Efficiency:** పెద్ద ప్రాజెక్ట్స్ చేసేటప్పుడు Sass వాడితే పని చాలా వేగంగా అవుతుంది.

---

**అభినందనలు!** మీరు అడిగిన 16 ముఖ్యమైన CSS & Sass టాపిక్స్ అన్నీ పూర్తి చేశారు. మీరు ఇప్పుడు ఒక ప్రొఫెషనల్ వెబ్ డెవలపర్ లాగా స్టైలింగ్ చేయగలరు.

ఇప్పుడు మనం **CSS Architecture (BEM)** మరియు **Sass (Mixins)** ఉపయోగించి ఒక రియల్ వరల్డ్ ప్రాజెక్ట్ (ఉదా: ఒక **Dashboard UI** లేదా **Portfolio Landing Page**) డిజైన్ చేద్దామా? దేనితో మొదలుపెడదాం?

---

# 17. Frameworks

CSS ఫ్రేమ్‌వర్క్స్ అనేవి మన పనిని సులభతరం చేస్తాయి. ప్రతిసారీ సున్నా నుండి స్టైల్స్ రాయకుండా, ఇవి ముందే తయారు చేసిన (Pre-built) కోడ్‌ని ఇస్తాయి. వీటిని ఉపయోగించి మనం చాలా వేగంగా వెబ్‌సైట్‌లను డెవలప్ చేయవచ్చు.

ప్రధానమైన నాలుగు ఫ్రేమ్‌వర్క్స్ గురించి చూద్దాం:

---

### 1. Bootstrap

ఇది చాలా పాతది మరియు అందరికీ తెలిసిన ఫ్రేమ్‌వర్క్.

- **ప్రత్యేకత:** ఇందులో ముందే తయారు చేసిన కాంపోనెంట్స్ (Navbar, Cards, Buttons, Modals) ఉంటాయి.
- **ఎలా వాడాలి:** HTML లో కేవలం క్లాస్ నేమ్స్ (ఉదా: `class="btn btn-primary"`) వాడితే చాలు, స్టైలింగ్ ఆటోమేటిక్‌గా వచ్చేస్తుంది.
- **ఎవరికి:** వేగంగా ప్రాజెక్ట్ పూర్తి చేయాలనుకునే వారికి మరియు డిజైన్ గురించి పెద్దగా ఆలోచించని వారికి ఇది బెస్ట్.

### 2. Tailwind CSS

ప్రస్తుతం సాఫ్ట్‌వేర్ రంగంలో ఇది ట్రెండింగ్‌లో ఉంది.

- **ప్రత్యేకత:** ఇది **Utility-first** ఫ్రేమ్‌వర్క్. అంటే ఇందులో `btn` లాంటి కాంపోనెంట్స్ ఉండవు, ప్రతి ప్రాపర్టీకి ఒక చిన్న క్లాస్ ఉంటుంది (ఉదా: `flex`, `pt-4`, `text-center`, `rounded-lg`).
- **లాభం:** మీరు CSS ఫైల్స్ రాయకుండానే HTML లోపలే కస్టమ్ డిజైన్స్ చేసుకోవచ్చు. దీనివల్ల వెబ్‌సైట్ సైజ్ కూడా చాలా తక్కువ ఉంటుంది.
- **ఎవరికి:** పూర్తి కస్టమ్ డిజైన్ (Unique look) కావాలనుకునే డెవలపర్లకు ఇది బెస్ట్.

### 3. Bulma

ఇది పూర్తిగా **Flexbox** మీద ఆధారపడిన మోడ్రన్ ఫ్రేమ్‌వర్క్.

- **ప్రత్యేకత:** ఇది చాలా తేలికగా (Lightweight) ఉంటుంది. ఇందులో JavaScript అస్సలు ఉండదు (కేవలం CSS మాత్రమే).
- **ఎలా ఉంటుంది:** దీని క్లాస్ నేమ్స్ చాలా సహజంగా ఉంటాయి (ఉదా: `class="button is-primary"`).
- **ఎవరికి:** సింపుల్‌గా, అందమైన డిజైన్ తక్కువ కోడ్‌తో చేయాలనుకునే వారికి.

### 4. Foundation

ఇది చాలా పెద్ద మరియు పవర్ ఫుల్ ఫ్రేమ్‌వర్క్.

- **ప్రత్యేకత:** ఇది చాలా కాంప్లెక్స్ వెబ్ అప్లికేషన్ల కోసం, ఈ-కామర్స్ సైట్ల కోసం బాగా ఉపయోగపడుతుంది.
- **ఎవరికి:** పెద్ద టీమ్‌లు కలిసి పని చేసే భారీ ప్రాజెక్టులకు ఇది అనుకూలం.

---

### క్లుప్తంగా కంపారిజన్ (Comparison Table):

| ఫ్రేమ్‌వర్క్   | రకం              | ప్రధాన బలం                              |
| -------------- | ---------------- | --------------------------------------- |
| **Bootstrap**  | Component-based  | సులభంగా అర్థమవుతుంది, వాడటం చాలా తేలిక. |
| **Tailwind**   | Utility-based    | పూర్తి కస్టమైజేషన్, వేగవంతమైన డిజైన్.   |
| **Bulma**      | Flexbox-based    | చాలా క్లీన్ కోడ్, CSS మాత్రమే.          |
| **Foundation** | Enterprise-level | చాలా పెద్ద ప్రాజెక్టులకు అనువైనది.      |

---

### నా సలహా:

1. మీరు ఇప్పుడే నేర్చుకుంటుంటే **Bootstrap** ట్రై చేయండి, ఇది కాన్సెప్ట్ అర్థం చేసుకోవడానికి చాలా సులభం.
2. మీరు ప్రొఫెషనల్‌గా ఎదగాలనుకుంటే, ప్రస్తుతం కంపెనీలు ఎక్కువగా **Tailwind CSS** వాడుతున్నాయి, కాబట్టి దానిని నేర్చుకోవడం చాలా మంచిది.

**అభినందనలు!** మీరు CSS లోని అన్ని కీలకమైన టాపిక్స్ (Fundamentals నుండి Frameworks వరకు) పూర్తి చేశారు! మీరు ఇప్పుడు ఒక వెబ్ డెవలపర్ గా మీ ప్రయాణాన్ని మొదలుపెట్టడానికి సిద్ధంగా ఉన్నారు.

ఇప్పుడు మనం నేర్చుకున్న ఈ నాలెడ్జ్‌తో ఒక **"Portfolio Website"** ని ఎలా నిర్మించాలో ప్లాన్ చేద్దామా? లేదా ఈ ఫ్రేమ్‌వర్క్స్‌లో ఏదైనా ఒకటి ప్రాక్టికల్ గా చేసి చూద్దామా?

---

# 18. Projects

మీరు CSS లోని అన్ని ముఖ్యమైన అంశాలను, బేసిక్స్ నుండి ఫ్రేమ్‌వర్క్స్ వరకు అద్భుతంగా పూర్తి చేశారు. ఇప్పుడు మనం నేర్చుకున్నదంతా ఆచరణలో పెట్టే సమయం వచ్చింది. ఈ ప్రాజెక్ట్‌లను ఒక క్రమ పద్ధతిలో చేద్దాం, దీనివల్ల మీకు "రియల్-వరల్డ్" ఎక్స్‌పీరియన్స్ వస్తుంది.

మీ కెరీర్‌కు ఉపయోగపడేలా, ఈ 6 ప్రాజెక్టులను నేను ఇలా ప్లాన్ చేస్తున్నాను:

### 1. Landing Page (లేఅవుట్ ప్రాక్టీస్)

- **ఏమి నేర్చుకుంటారు:** Flexbox, Grid, Media Queries, Responsive Images.
- **టార్గెట్:** హెడర్, హీరో సెక్షన్ (బ్యాక్‌గ్రౌండ్ ఇమేజ్ + టెక్స్ట్), ఫుటర్.

### 2. Portfolio Website (స్టైలింగ్ & యానిమేషన్)

- **ఏమి నేర్చుకుంటారు:** CSS Transitions, Transforms, Animations, Google Fonts, Box Model.
- **టార్గెట్:** అబౌట్ మీ సెక్షన్, ప్రాజెక్ట్ కార్డ్స్ (Hover effects), కాంటాక్ట్ ఫామ్.

### 3. Responsive Blog (టెక్స్ట్ & టైపోగ్రఫీ)

- **ఏమి నేర్చుకుంటారు:** Line-height, Letter-spacing, Advanced Selectors, Responsive Grid.
- **టార్గెట్:** చదవడానికి అనువైన టైపోగ్రఫీ, సైడ్‌బార్, ఆర్టికల్ రీడబిలిటీ.

### 4. Dashboard UI (గ్రిడ్ & కాంపోనెంట్స్)

- **ఏమి నేర్చుకుంటారు:** CSS Grid, Custom Scrollbars, CSS Variables.
- **టార్గెట్:** సైడ్ నావిగేషన్ మెనూ, స్టాటిస్టిక్స్ కార్డ్స్, డేటా టేబుల్స్.

### 5. E-commerce UI (అడ్వాన్స్‌డ్ ఎఫెక్ట్స్)

- **ఏమి నేర్చుకుంటారు:** Filter/Backdrop-filter, Object-fit, Aspect-ratio.
- **టార్గెట్:** ప్రొడక్ట్ గ్రిడ్, కార్డ్ గ్లాస్-ఎఫెక్ట్, ప్రైస్ ట్యాగ్స్.

### 6. Admin Panel (ఫ్రేమ్‌వర్క్ ప్రాక్టీస్)

- **ఏమి నేర్చుకుంటారు:** Tailwind CSS లేదా Bootstrap ఫ్రేమ్‌వర్క్, BEM ఆర్కిటెక్చర్.
- **టార్గెట్:** మోడ్రన్ డ్యాష్‌బోర్డ్, మోడల్స్, అలర్ట్స్, ఫామ్ వాలిడేషన్.

**మీరు సిద్ధంగా ఉంటే, చెప్పండి:**

2. మీరు కోడ్ రాయడానికి ఏ ఎడిటర్ (VS Code లాంటిది) వాడుతున్నారు?
